Externe Tools

Yass Reloaded lässt sich mit optionalen externen Tools für Vocal-Separation und Lyrics-Transkription verbinden. Öffne Einstellungen > Externe Tools zur Konfiguration.

FFmpeg

FFmpeg wird für die Audio-Konvertierung (AAC, OGG, OPUS usw.) und den Vocal-Separations-Workflow benötigt. Yass kann FFmpeg automatisch herunterladen, oder du gibst einen vorhandenen Pfad unter Externe Tools > Speicherorte an.

  • Download: https://ffmpeg.org/download.html
  • Unter Windows den Pfad zum bin-Ordner mit ffmpeg.exe angeben.

Python-Runtimes

Mehrere lokale Tools verwenden Python. Unter Externe Tools > Speicherorte kannst du ein Standard-Python-Executable festlegen. Yass verwendet dieses als gemeinsamen Fallback für leichte Python-Helfer und als Bootstrap-Interpreter zum Erzeugen tool-spezifischer Umgebungen.

  • Wenn ein tool-spezifisches Python-Feld leer bleibt, fällt Yass auf das Standard-Python aus Speicherorte zurück.
  • Wenn du bei WhisperX oder audio-separator auf Package aktualisieren klickst und noch keine tool-spezifische Python-Runtime gesetzt ist, erzeugt Yass automatisch eine eigene virtuelle Umgebung für dieses Tool und speichert dort den Python-Pfad.
  • So bleiben schwere Abhängigkeiten wie WhisperX und audio-separator isoliert, während kleinere Python-Helfer weiter das Standard-Python gemeinsam nutzen können.

yt-dlp

yt-dlp wird im Song-Erstellungs-Assistenten verwendet, um Audio von YouTube und anderen Plattformen herunterzuladen.

  • Installation: pip install yt-dlp oder als Standalone-Binary herunterladen.
  • Codec, Auflösung und Audioformat unter Einstellungen > Assistent > yt-dlp konfigurieren.

MVSEP (Online-Vocal-Separation)

MVSEP ist ein cloudbasierter Vocal-Separationsdienst. Das Audio wird hochgeladen, in Stems aufgeteilt und das Ergebnis heruntergeladen. Erfordert ein Konto und einen API-Token.

  1. Konto anlegen unter https://mvsep.com und API-Token aus dem Profil kopieren.
  2. Einstellungen > Externe Tools > MVSEP öffnen und Token einfügen.
  3. Separationsmodell wählen. Das Vokalmodell kann meist auf dem Standard bleiben, und das Audioformat legt das zurückgelieferte Stem-Format fest.
  4. Im Assistenten auf Vocals separieren + transkribieren klicken.
  5. Für bereits geöffnete Songs: Bearbeiten > Audio separieren.

audio-separator (lokale Vocal-Separation)

audio-separator führt vortrainierte Separationsmodelle lokal aus — ohne Internet oder API-Schlüssel. Es unterstützt dieselben MDX-Net-, VR-Architecture- und Demucs-Modelle wie Ultimate Vocal Remover (UVR).

  1. Python 3.9 oder neuer installieren.
  2. audio-separator installieren:
    • Nur CPU: pip install audio-separator[cpu]
    • NVIDIA GPU: pip install audio-separator[gpu]
  3. Einstellungen > Externe Tools > audio-separator öffnen.
  4. Das Python-Executable für dieses Tool angeben oder leer lassen, damit das Standard-Python aus Externe Tools > Speicherorte verwendet wird.
  5. Wenn du Package aktualisieren bei leerem tool-spezifischen Python-Feld benutzt, erstellt Yass automatisch eine eigene virtuelle audio-separator-Umgebung und trägt sie dort ein.
  6. Modell auswählen. Bekannte Modelle werden mit lesbarem Namen angezeigt; unbekannte Modelle erscheinen mit ihrem Dateinamen. Empfehlung:
    • Kim_Vocal_2.onnx - schnell, gute Qualität (Standard)
    • MDX23C-8KFFT-InstVoc_HQ.ckpt - hohe Qualität
    • BS-Roformer-Viperx-1297.ckpt - höchste Qualität, langsamer
  7. Optional: Modell-Cache-Verzeichnis angeben. Dort speichert audio-separator heruntergeladene Modelle, damit sie nicht erneut geladen werden müssen.
  8. Ausgabeformat steuert das rohe Stem-Format, das audio-separator zuerst schreibt, bevor Yass die Dateien in dein konfiguriertes Zielformat konvertiert.
  9. Konfiguration testen klicken, um die Einrichtung zu prüfen.
  10. Wenn der Systemtest erfolgreich war, wird die Schaltfläche Vocals separieren + transkribieren im Assistenten aktiviert. audio-separator hat Vorrang vor MVSEP, wenn beide konfiguriert sind.

OpenAI Whisper (Cloud-Transkription)

Die OpenAI Whisper API transkribiert Audio mit Zeitstempeln und überträgt die Ergebnisse auf Noten. Erfordert einen OpenAI API-Schlüssel.

  1. API-Schlüssel unter https://platform.openai.com/api-keys erstellen.
  2. Einstellungen > Externe Tools > OpenAI öffnen und Schlüssel eintragen.
  3. Transkriptionsmodell (z.B. whisper-1) und optional einen Sprachcode wählen.
  4. Im Editor: Bearbeiten > Noten ausrichten (Transkription).

WhisperX (lokale Transkription)

WhisperX führt Whisper-basierte Transkription lokal mit wortgenauen Zeitstempeln aus. Kein Internet erforderlich.

  1. Python 3.9 oder neuer installieren.
  2. WhisperX installieren: pip install whisperx
  3. Einstellungen > Externe Tools > WhisperX öffnen.
  4. Das Python-Executable für WhisperX angeben oder leer lassen, damit das Standard-Python aus Externe Tools > Speicherorte verwendet wird.
  5. Wenn du Package aktualisieren bei leerem WhisperX-Python-Feld benutzt, erstellt Yass automatisch eine eigene virtuelle WhisperX-Umgebung und speichert diese Runtime.
  6. Aufrufmodus wählen (python -m whisperx empfohlen).
  7. Modell (z.B. large-v2) und Gerät (cpu oder cuda) wählen.
  8. Konfiguration testen klicken. Das Ergebnis wird gespeichert und aktiviert die Vocals separieren + transkribieren-Schaltfläche im Assistenten.
  9. Im Editor: Bearbeiten > Noten ausrichten (Transkription), dann WhisperX wählen.

Song-Assistent: vollständige Pipeline

Der Assistent kann die vollständige Pipeline automatisch ausführen:

  1. YouTube-URL auf der YouTube-Seite eingeben und Audio herunterladen.
  2. Auf der Liedtext-Seite auf Vocals separieren + transkribieren klicken.
  3. Yass konvertiert das Audio zu WAV, trennt die Vocals (audio-separator oder MVSEP) und transkribiert sie (WhisperX).
  4. Der Liedtext wird automatisch eingefügt. Du kannst ihn vor dem Fortfahren korrigieren.
  5. Assistenten abschließen — der Song wird mit vorausgefüllten Zeitstempeln aus der Transkription erstellt.

Voraussetzungen: audio-separator (lokal, Systemtest bestanden) oder MVSEP (API-Token konfiguriert), sowie WhisperX (Systemtest bestanden).
