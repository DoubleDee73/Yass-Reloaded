Narzędzia zewnętrzne

Yass Reloaded integruje się z kilkoma opcjonalnymi narzędziami zewnętrznymi do separacji wokali i transkrypcji tekstu. Otwórz Ustawienia > Narzędzia zewnętrzne, aby je skonfigurować.

FFmpeg

FFmpeg jest wymagany do konwersji audio (AAC, OGG, OPUS itd.) oraz do procesu separacji wokali. Yass może pobrać FFmpeg automatycznie albo możesz wskazać istniejącą instalację w Narzędzia zewnętrzne > Lokalizacje.

  • Pobierz: https://ffmpeg.org/download.html
  • W systemie Windows wskaż folder bin zawierający ffmpeg.exe.

Środowiska Python

Kilka lokalnych narzędzi używa Pythona. W Narzędzia zewnętrzne > Lokalizacje możesz ustawić domyślny executable Pythona. Yass używa go jako wspólnego fallbacku dla lekkich pomocników Pythonowych oraz jako interpretera bazowego do tworzenia środowisk specyficznych dla poszczególnych narzędzi.

  • Jeśli pole Python dla danego narzędzia jest puste, Yass użyje domyślnego Pythona z Lokalizacji.
  • Jeśli klikniesz Update package dla WhisperX lub audio-separator, a pole Python dla narzędzia jest puste, Yass automatycznie utworzy dedykowane środowisko wirtualne i zapisze tam ścieżkę do Pythona.
  • Dzięki temu ciężkie zależności, takie jak WhisperX i audio-separator, pozostają odizolowane, a mniejsze helpery Pythonowe mogą współdzielić domyślnego Pythona.

yt-dlp

yt-dlp jest używany przez kreator tworzenia piosenek do pobierania audio z YouTube i innych platform. Nie wymaga klucza API.

  • Instalacja: pip install yt-dlp albo pobranie samodzielnego pliku binarnego.
  • Skonfiguruj kodek, rozdzielczość i format audio w Ustawienia > Kreator > yt-dlp.

MVSEP (separacja wokali online)

MVSEP to chmurowa usługa separacji wokali. Wysyła audio do API MVSEP, rozdziela je na stemsy i pobiera wynik. Wymaga konta i tokenu API.

  1. Zarejestruj się na https://mvsep.com i skopiuj token API ze swojego profilu.
  2. Otwórz Ustawienia > Narzędzia zewnętrzne > MVSEP i wklej token.
  3. Wybierz model separacji. Model wokalny zwykle może pozostać na wartości domyślnej, a Format audio określa format zwracanych stemów.
  4. W kreatorze kliknij Separate Vocals + Transcribe, aby uruchomić cały pipeline.
  5. Dla piosenek już otwartych w edytorze użyj Edycja > Separate Audio.

audio-separator (lokalna separacja wokali)

audio-separator uruchamia wytrenowane modele separacji lokalnie na Twoim komputerze, bez internetu i bez klucza API. Obsługuje te same modele MDX-Net, VR Architecture i Demucs co Ultimate Vocal Remover (UVR).

  1. Zainstaluj Python 3.9 lub nowszy.
  2. Zainstaluj audio-separator:
    • Tylko CPU: pip install audio-separator[cpu]
    • GPU NVIDIA: pip install audio-separator[gpu]
  3. Otwórz Ustawienia > Narzędzia zewnętrzne > audio-separator.
  4. Wskaż executable Pythona dla tego narzędzia albo zostaw pole puste, aby użyć domyślnego Pythona z Lokalizacji.
  5. Jeśli użyjesz Update package przy pustym polu Python dla narzędzia, Yass utworzy dedykowane środowisko wirtualne dla audio-separator i automatycznie uzupełni to pole.
  6. Wybierz model. Znane modele są pokazywane czytelną nazwą, a nieznane pozostają przy nazwie pliku. Polecane modele:
    • Kim_Vocal_2.onnx - szybki, dobra jakość (domyślny)
    • MDX23C-8KFFT-InstVoc_HQ.ckpt - wysoka jakość
    • UVR-MDX-NET-Inst_HQ_3.onnx - alternatywa
    • BS-Roformer-Viperx-1297.ckpt - najwyższa jakość, wolniejszy
  7. Opcjonalnie ustaw katalog pamięci podręcznej modeli. Pobrane modele będą tam przechowywane, aby nie trzeba było pobierać ich ponownie.
  8. Output format określa surowy format plików zapisywanych przez audio-separator, zanim Yass przekonwertuje stemsy do skonfigurowanego formatu docelowego.
  9. Kliknij Test configuration, aby sprawdzić konfigurację.
  10. Po pomyślnym teście przycisk Separate Vocals + Transcribe w kreatorze stanie się dostępny. Jeśli skonfigurowane są zarówno MVSEP, jak i audio-separator, preferowany będzie audio-separator.

OpenAI Whisper (transkrypcja w chmurze)

API OpenAI Whisper transkrybuje audio z oznaczeniami czasu dla słów i przenosi wynik na nuty w edytorze. Wymaga klucza API OpenAI.

  1. Pobierz klucz API na https://platform.openai.com/api-keys.
  2. Otwórz Ustawienia > Narzędzia zewnętrzne > OpenAI i wpisz klucz.
  3. Wybierz model transkrypcji (np. whisper-1) oraz opcjonalnie kod języka.
  4. W edytorze użyj Edycja > Align Notes (Transcription), aby transkrybować i wyrównać aktualną piosenkę.

WhisperX (transkrypcja lokalna)

WhisperX uruchamia lokalnie transkrypcję opartą na Whisper z timestampami na poziomie słowa. Nie wymaga internetu ani klucza API.

  1. Zainstaluj Python 3.9 lub nowszy.
  2. Zainstaluj WhisperX: pip install whisperx
  3. Otwórz Ustawienia > Narzędzia zewnętrzne > WhisperX.
  4. Wskaż executable Pythona dla WhisperX albo zostaw pole puste, aby użyć domyślnego Pythona z Lokalizacji.
  5. Jeśli użyjesz Update package przy pustym polu Python dla WhisperX, Yass automatycznie utworzy dedykowane środowisko wirtualne i zapisze tę runtime.
  6. Wybierz tryb uruchamiania (python -m whisperx jest zalecane).
  7. Wybierz model (np. large-v2) i urządzenie (cpu albo cuda).
  8. Kliknij Test configuration, aby uruchomić test. Wynik zostanie zapisany i włączy przycisk Separate Vocals + Transcribe w kreatorze.
  9. W edytorze użyj Edycja > Align Notes (Transcription) i wybierz WhisperX.

Kreator tworzenia piosenki: pełny pipeline

Kreator może automatycznie uruchomić cały pipeline:

  1. Wpisz adres YouTube na stronie YouTube i pobierz audio.
  2. Na stronie Lyrics kliknij Separate Vocals + Transcribe.
  3. Yass konwertuje audio do WAV, oddziela wokale (audio-separator albo MVSEP) i transkrybuje je (WhisperX).
  4. Tekst zostanie wypełniony automatycznie. Możesz go poprawić przed kontynuacją.
  5. Zakończ kreator, aby utworzyć piosenkę z już wypełnionymi timestampami z transkrypcji.

Wymagania: audio-separator (lokalnie, test zakończony sukcesem) albo MVSEP (skonfigurowany token API), oraz WhisperX (test zakończony sukcesem).
